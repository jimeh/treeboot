Copy and Sync Metadata Preservation

Plan for making Treeboot preserve file and directory permissions, owner, and group by default during copy and sync, while adding small opt-outs for projects that want less metadata repair.

Contents Decision Contract Config and CLI Sync Semantics Implementation Testing Rollout Resolved Choices

Decision

Recommended path: improve the default behavior and add opt-outs. Do not add hard-link support. Do not add archive-style ACL, xattr, resource-fork, file-flag, or sparse-file preservation.

Current Treeboot already preserves regular-file contents, permissions, and modified time when it copies or replaces a file. The new work should make the contract broader and more explicit: default copy and sync should also preserve owner and group where the platform supports them, and sync should detect and repair metadata drift without needing content drift.

The feature should remain narrow because Treeboot is a worktree bootstrapper, not a backup/archive tool. Projects that need exact archive behavior should use project-local commands such as rsync, cp -a, or platform-specific tools in configured commands.

Contract

Default Metadata

  • Regular files preserve contents, permissions, owner, group, and modified time.
  • Directories preserve permissions, owner, and group when they are created or repaired by sync.
  • Directory modified time is not preserved or compared.
  • Source symlink behavior stays Treeboot-specific and safe, using the existing symlink preservation and rewrite rules.

Non-Goals

  • No hard-link support in this milestone.
  • No ACL, extended attribute, resource fork, file flag, or sparse-file layout preservation.
  • No owner/group support on platforms where Rust cannot expose or set ownership safely.
  • No implicit fallback to external commands.
Metadata Default Opt-out Sync drift detection Failure behavior
Permissions Preserve for regular files and directories. ignore_metadata = ["permissions"] Detect and repair unless ignored. Hard error for unexpected apply failures.
Owner Attempt preservation on Unix-like platforms. ignore_metadata = ["owner"] or ["ownership"] Detect and attempt repair unless ignored. Permission-denied repair failure warns and continues; unexpected errors fail.
Group Attempt preservation on Unix-like platforms. ignore_metadata = ["group"] or ["ownership"] Detect and attempt repair unless ignored. Permission-denied repair failure warns and continues; unexpected errors fail.
Modified time Preserve for regular files. None in this milestone. Continue to drive default file content comparison. Hard error for unexpected apply failures.

Ownership caveat: normal users often cannot set arbitrary owner or group values. The default should still attempt ownership preservation, but permission-denied ownership repair must not make a normal bootstrap unusable.

Config and CLI

Add one opt-out field to copy and sync entries. It should be invalid for symlink entries. 

copy = [
  { source = ".env.local", ignore_metadata = ["ownership"] },
]

sync = [
  { source = "shared/config", ignore_metadata = ["permissions"] },
  { source = "shared/state", ignore_metadata = ["owner", "group"] },
]

files = [
  { operation = "copy", source = ".npmrc", ignore_metadata = ["ownership"] },
  { operation = "sync", source = "shared/editor", ignore_metadata = ["permissions"] },
]

Allowed Values

  • permissions: do not compare or apply permission metadata.
  • owner: do not compare or apply owner metadata.
  • group: do not compare or apply group metadata.
  • ownership: shorthand for owner plus group.

Manual commands should expose the same policy with a repeated flag: 

treeboot copy .env.local --ignore-metadata ownership
treeboot sync shared/config --ignore-metadata permissions
treeboot sync shared/state --ignore-metadata owner --ignore-metadata group

The CLI can also accept comma-separated values if that fits the existing clap style cleanly, but repeated flags are the required behavior. Completion should list the allowed values.

Sync Semantics

Sync should compare content and the configured metadata set. If only metadata differs, Treeboot should repair metadata in place instead of replacing the file contents.

Source and target state Default behavior With ignore option Output
File bytes differ. Replace target file, then apply metadata. Apply all non-ignored metadata after replacement. treeboot: sync source -> target
Only file permissions differ. Repair permissions in place. No action when permissions are ignored. treeboot: sync metadata source -> target
Only owner or group differs. Attempt ownership repair in place. No action when owner/group is ignored. treeboot: sync metadata source -> target, plus warning on permission denied.
Directory permissions or ownership differ. Repair directory metadata during sync traversal. No action for ignored fields. treeboot: sync metadata source -> target
Directory mtime differs. No action. No action. No output.

Compare Modes

compare = "metadata" should continue to use size and modified time for regular-file content drift, then also compare the configured metadata fields. compare = "checksum" should use content comparison for bytes, then still compare the configured metadata fields. The ignore_metadata field controls only metadata comparison and repair, not content comparison.

Implementation

  1. Spec first. Update docs/SPEC.md with the new metadata contract, manual flag, config field, output examples, and platform caveats. Bump the visible spec version from v1.9.0 to v1.10.0 and keep the README spec reference in sync if present.
  2. Add public policy types in treeboot-core. Suggested shape: 
    pub enum MetadataField {
    Permissions,
    Owner,
    Group,
    Ownership,
}

pub struct MetadataPolicy {
    pub permissions: bool,
    pub owner: bool,
    pub group: bool,
}
    Store the normalized policy on FileOperation and PlannedFileOperation. Keep public APIs documented.
  3. Parse ignore_metadata in config.rs. Reject it for symlink entries through the existing invalid-field path. Expand ownership to owner plus group during normalization.
  4. Add --ignore-metadata to manual copy and sync commands in the CLI crate. Reject or omit it for symlink. Pass the normalized policy through ManualFileOperationOptions and FileOperationOptions.
  5. Update schema generation in crates/treeboot-core/examples/generate_config_schema.rs and regenerate both checked-in schema assets with mise run generate.
  6. Split metadata operations in files.rs:
    • Add a metadata comparison helper for files and directories.
    • Add a metadata apply helper that can apply permissions, owner, and group according to policy.
    • Add a FileAction::RepairMetadata action so sync can repair metadata-only drift without rewriting bytes.
    • Keep file replacement applying mtime before permissions so read-only copied files remain supported.
  7. Use the standard library ownership APIs on Unix. Rust 1.96 exposes std::os::unix::fs::chown, fchown, and lchown, so this should not require unsafe code or a new runtime dependency. Keep Windows owner and group behavior unsupported and documented.
  8. Add ownership warning output. Prefer a structured OutputEvent variant over string-only reporting so JSON or future structured output remains coherent.
  9. Refresh text output and dry-run output: 
    treeboot: sync metadata shared/config -> shared/config
treeboot: would sync metadata shared/config -> shared/config
treeboot: warning: could not preserve ownership shared/config: operation not permitted

Pre-Handoff Gate

For the implementation PR, run mise run check. If the change touches release metadata, workflows, or broader architecture docs beyond this plan, run mise run verify.

Rollout

  1. Land spec and docs in the same implementation branch as code so the contract and behavior cannot drift.
  2. Keep README short. Mention the default ownership preservation and link to the spec instead of duplicating edge-case matrices.
  3. Regenerate schema assets before review.
  4. Call out the behavior change in the PR: sync may now report and repair permission or ownership drift even when file bytes match.
  5. Call out that ownership preservation is best-effort when the OS denies owner/group changes.

Resolved Choices

  1. Permission repair failures stay operation failures. Only owner/group permission-denied failures are best-effort warnings.
  2. ignore_metadata = ["ownership"] is accepted as input shorthand and normalized to ["owner", "group"] in treeboot config output.
  3. Metadata-only sync uses treeboot: sync metadata source -> target and dry-run uses treeboot: would sync metadata source -> target.
